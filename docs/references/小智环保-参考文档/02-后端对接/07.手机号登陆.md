# 请求登录 (本地手机号登录)

本接口适用于手机号登录,当用户在屏幕上输入手机号后，设备优先尝试使用本地的用户（会员）信息， 如果本地不存在，将向后端请求用户信息，后端需要按返回正确的用户（会员）结构信息方可登录设备

后续步骤：如果需要禁用某个离线的用户，即已存储在设备上的用户，则需要调用清除用户信息接口进行清除。

请求路径：

    GET {{baseUrl}}/login_phone?phone=18888****88    // v7.65- （弃用）
    POST {{baseUrl}}/login_phone     // v7.65+

请求参数：v7.65- （弃用）

| 序号 | 字段  | 说明                                 |
|:-----|:------|:-------------------------------------|
| 1    | phone | 手机号 11位纯数字 ,phone拼接在路径中 |

请求body：v7.65+

| 序号 | 字段  | 长度 | 必填 | 说明              | 示例            |
|:-----|:------|:-----|:-----|:------------------|:----------------|
| 1    | phone | 11   | 是   | 手机号 11位纯数字 | 18888\*\*\*\*88 |

后端应返回：

| 序号 | 字段 | 值       | 说明                              |
|:-----|:-----|:---------|:----------------------------------|
| 1    | code | 0        | 必须为0才会识别为正确的结果       |
| 2    | data | 会员信息 | 见附件【member】表                |
| 3    | msg  | 文本     | 可选，错误提示 ，用于跟踪错误信息 |

其中data 见【member】表实体，必填信息如下，

| 序号 | 字段       | 必填 | 说明                                         |
|:-----|:-----------|:-----|:---------------------------------------------|
| 1    | userId     | 是   | 用户ID                                       |
| 2    | loginName  |      | 登录名称、IC或ID卡号                         |
| 3    | phone      | 是   | 手机号                                       |
| 4    | role       | 是   | 角色 清用员：4，普通用户:9                   |
| 5    | status     | 是   | 是否禁用状态，大于0表示禁用                  |
| 6    | avatar     |      | 头像照片路径                                 |
| 7    | phoneLogin |      | 是否可以使用手机号登录，2：不允许， 其它允许 |

后端返回示例：

``` json
{
    "code":0,
    "data":{  // 见【member】表实体
       "userId":123,
       "loginName":"root", //登录名、IC或ID卡号
        "phone":"18888888888",
        "role":9,
        "status":0,// 是否禁用，大于0表示禁用   
        "avatar":"https://resources.zsgai.com/assets/img/hbws.png"
    },
    "msg":"查询成功"
}
```
