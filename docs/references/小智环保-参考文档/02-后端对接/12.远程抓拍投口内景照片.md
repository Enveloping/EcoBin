# 远程抓拍投口内景（USB相机）照片

支持版本： v8.15.12+

此命令用于自定义内景照片上传，（不同与投递和清运等业务的内景照片上传）此接口调用频率间隔至少2.5秒， 使用前先在平台配置的照片接收服务器地址，并下发到设备。

请求路径：

    POST https://device.zsgai.com/open_api/remote_capture_door_photo

请求参数：

1、 header：

| 序号 | 字段 | 使用场景 |
|:---|:---|:---|
| 1 | `Jnx-Access-Token` | 客户端请求时将`id_token`作为`Jnx-Access-Token`传递到服务端 |
| 2 | `Content-Type` | application/json |

2、body:

| 序号 | 字段 | 数据类型 | 非空 | 默认值 | 说明 |
|:---|:---|:---|:---|:---|:---|
| 1 | devSn | String | 必填 |  | 设备编号 |
| 2 | sn | String | 可选 |  | 与订单编号相同 用于跟踪日志，长度固定32位，不传平台可自动生成 |
| 3 | doorId | Integer | 可选 |  | 投口编号 不传默认为1 |

返回：

| 序号 | 字段 | 使用场景 |
|:---|:---|:---|
| 1 | `code` | 0 成功 -1 失败 成功只是带表命令下发成功，具体结果需要上传日志或根据接收接口进行跟踪 |
| 2 | `msg` | 错误提示消息 |

3、设备上传USB内景照片

接收姿势与投递内景照片相同，只是路径不同

请求路径：

    POST {{baseUrl}}/capture_door_photo  # USB内景照片上传路径 

请求参数：

1、 header：

| 序号 | 字段           | 使用场景         |
|:-----|:---------------|:-----------------|
| 1    | `Content-Type` | application/json |

2、 params/FormData：

| 序号 | 字段     | 类型                        | 必填 | 默认值 | 说明 |
|:-----|:---------|:----------------------------|------|--------|------|
| 1    | sn       | 订单编号 下发命令时的单据号 |      |        |      |
| 2    | type1    | 8 固定值                    |      |        |      |
| 3    | type2    | 8 固定值                    |      |        |      |
| 4    | type3    | 1 固定值                    |      |        |      |
| 5    | 公共参数 | 参见 【附件\>公共参数】     |      |        |      |

3、 body： 具体的照片信息

| 序号 | 字段 | 类型                     |
|:-----|:-----|:-------------------------|
| 1    | file | application/octet-stream |

需要返回： http状态码为200，同时返回的数据中code=0，正确返回后会立即删除本地照片以释放存储空间，

| 序号 | 字段   | 使用场景       |
|:-----|:-------|:---------------|
| 1    | `code` | 0 成功 -1 失败 |
| 2    | `msg`  | 错误提示消息   |

后端返回示例：

``` json
{
    "code":0,
    "msg":"上传成功"
}
```
