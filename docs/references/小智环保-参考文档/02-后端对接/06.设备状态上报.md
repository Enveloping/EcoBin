# 设备状态上报

当开启第三方对接，或只上报状态到第三方时，参见此文档进行状态接收，上报的频繁可以设备配置中进行调整，默认间隔时间为60秒一次。

请求路径：

    POST {{baseUrl}}/device_state

请求参数：

1、 header：

| 序号 | 字段           | 使用场景         |
|:-----|:---------------|:-----------------|
| 1    | `Content-Type` | application/json |

2、 params：

| 序号 | 字段     | 类型                    | 必填 | 默认值 | 说明 |
|:-----|:---------|:------------------------|------|--------|------|
| 1    | 公共参数 | 参见 【附件\>公共参数】 |      |        |      |

3、body:

| 序号 | 参数             | 值                              |
|:-----|:-----------------|:--------------------------------|
| 1    | allSmoke         | 全局烟雾报警 0未报警 1报警      |
| 2    | isSpill          | 是否满溢 0未满溢 1满溢          |
| 3    | weightSpill      | 是否超重报警 0未报警 1报警      |
| 4    | compressdSpill   | 是否压缩报警 0未报警 1报警      |
| 5    | weightSys        | 当前设备理论重量，多投口累加 kg |
| 6    | weightNow        | 当前设备即时重量，多投口累加 kg |
| 7    | size             | 投口的数量                      |
| 8    | appConfigVersion | 配置的版本号                    |
| 9    | listDoor         | 投口的状态列表 见附件【door】表 |
| 10   | stopService      | 是否停止服务，0未停止 1已停止   |
| ...  | ...              | 见下面的上传示例                |

需要返回：

| 序号 | 字段   | 使用场景       |
|:-----|:-------|:---------------|
| 1    | `code` | 0 成功 -1 失败 |
| 2    | `msg`  | 错误提示消息   |

# 上传示例：

支持版本：v7.60+

``` json
{  // 设备的全局状态，即：合并下面每个投口的状态

  "devId": 490 // 设备编号
  "appConfigVersion": 982, //设备的当前配置版本，在平台每生成一次自增1
  "weightSpill": 0,// 重量报警 0未报警 1报警
  "weightSys": 0,// 理论重量
  "size": 2,// 投口数量 
  "stopService": 0,// 停止服务状态，0未停止 1 已停止
  "compressedSpill": 0,// 压缩报警 0不报警 1报警，当正在压缩时，为报警 

  "bugInf": {},  // 当前的故障代码
  "voltage": 0, // 设备电压
  "weightNow": 1,// 即时重量 
  "appVersion": 760, // 设备版本
  "isSpill": 1,// 根据投口状态综合判断是否报警 0未报警 1报警
  "smokeSpill": 0,// 烟雾报警 0未报警 1报警
  "orgId": 3, // 对应平台的机构ID
  "deviceOff": false,//  0未停止 1 已停止 设备暂不可用 部分设备可在本地配置页面设置，表示从设备上强制关闭设备，不受后台控制
  "redSpill": 1,// 红外报警 0未报警 1报警
  "thirdApiOn": true, // 对接开关 
  "dataFrom": 1, // 数据来源(1:设备累加 2:设备即时重量)  v7.68
  "listDoor": [ // 投口列表
    {
      "weightSpill": 0,// 重量报警 0未报警 1报警
      "weightSys": 0,// 理论重量 多数以此值为准，但可能受硬件的稳定因素影响，会与即时重量`weightNow`有点差别，如变化较大，需要重新校准硬件
      "temper": 0, //    温度
      // "compressNum": 0,//已弃用，当前已压缩的次数 同compress.connt
      "humidity": 0, //    湿度
      "weight": 0, //    上一次重量
      "cleanDoor": 0,// 清运门状态 0关闭 1打开  
      "weightNow": 0,// 即时重量
      "compress": {  // 压缩机状态
        "connt": 1, //当前已压缩的次数 
        "maxConnt": 6,//最大允许压缩的次数 
        "percentage": 14,//压缩进度 %  空闲状态为0 ， 当有任务执行时，已完成的百分比，不支持置0，单位%
        "usedtime": 29,//已用时间 秒
        "speed": 0,// 速率，当前压缩机设置的压缩速率 (厘米每秒或百分比每秒 ) 暂不可用
        "state": 1,//1正在压缩
        "pausetime": 0,//暂不可用
        "maxtime": 201,//最大需要的压缩时间 秒
        "direction": 1 //正向  0 空闲 1 正向 2反向
      },
      "isSpill": 1,//综合判断的是否报警，即任一种情况报警此值为1，不报警为0
      "lastCleanTime": 1739966618000,//最后一次清运时间
      "smokeSpill": 0,// 烟雾报警 0未报警 1报警
      "redSpill": 1,// 红外报警 0未报警 1报警
      "doorId": 2,// 投口编号
      "compressdSpill": 0, // 压缩报警 0不报警 1报警，当正在压缩时，为报警 
      "deliveryDoor": 0 , // 投递门状态 0关闭 1打开

      //下面6个字段v7.61支持 
      "type1": 0 , // 一级分类 
      "type2": 0 , // 二级分类 
      "price": 0 , // 投递单价 元  
      "typePrice": 0 , //价格类型 默认1  
      "redirectDoorId": 0 , // 合并到的物理筒号 
      "merge": 0 ,  // 是否合并 0否 1是 不可用  
      "bagNo":"00000",   // 垃圾袋编码 v7.67
      "maxWeight":50.00   // 经过计算后的投口设置的最大重量(公斤)，来自机构、设备、投口的配置 ，优先取投口的值  v7.73
    },
    {
        ...
    }
  ],

  "enableSpill": true  // v7.61  满溢报警功能是否开启 true开启 false未开启 如需要临时禁用在后台点击消除满溢，重启后恢复，强制不开启请修改设备配置 
}
```
