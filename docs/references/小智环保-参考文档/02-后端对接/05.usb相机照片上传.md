# Usb 相机照片上传

当设备安装usb摄像头时，设备同时会调用内部的usb摄像头进行拍照， 设备usb摄像头拍照后会立即上传，图片数据上传，第三接收后，必须按照要求返回相应的信息 当发生断网或延时较高场景时，会启用离线分批次上传，为避免服务端压力过载，一次通常为1-2条，间隔1-10秒，直到上传完为止。 当拍照失败，或照片已在本地删除，则不再上传。

请求路径：

    POST {{baseUrl}}/updata_photo_delivery  # 投递照片上传路径
    POST {{baseUrl}}/updata_photo_clean     # 清运照片上传路径

请求参数：

1、 header：

| 序号 | 字段           | 使用场景         |
|:-----|:---------------|:-----------------|
| 1    | `Content-Type` | application/json |

2、 params/FormData：

| 序号 | 字段     | 类型                                   | 必填 | 默认值 | 说明 |
|:-----|:---------|:---------------------------------------|------|--------|------|
| 1    | sn       | 订单编号同订单中的sn,原`orderSn`已弃用 |      |        |      |
| 2    | type1    | 1投递 2清运 (弃用，通过接口区分)       |      |        |      |
| 3    | type2    | 1开门 2关门                            |      |        |      |
| 4    | type3    | 1内景 2外景                            |      |        |      |
| 5    | userId   | 用户id                                 |      |        |      |
| 6    | fileName | 保留                                   |      |        |      |
| 7    | filePath | 保留                                   |      |        |      |
| 8    | 公共参数 | 参见 【附件\>公共参数】                |      |        |      |

3、 body： 具体的照片信息

| 序号 | 字段 | 类型                     |
|:-----|:-----|:-------------------------|
| 1    | file | application/octet-stream |

需要返回：

| 序号 | 字段   | 使用场景       |
|:-----|:-------|:---------------|
| 1    | `code` | 0 成功 -1 失败 |
| 2    | `msg`  | 错误提示消息   |

后端返回示例：

``` json
{
    "code":0,
    "msg":"上传成功"
}
```
