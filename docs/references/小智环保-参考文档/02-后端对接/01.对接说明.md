# 对接说明

文档中所提到的`后端`为第三方服务器，`平台`为小智平台，

对接流程：创建应用、请求令牌、使用令牌请求相应接口，流程如下：

1、创建应用，获取`appid`和`secret`，注意：更换`secret`需要重新调用接口获取`id_token`、`refreshCode`

2、请求令牌，获取`id_token`、`refreshCode`信息

3、刷新令牌，使用`refreshCode`刷新令牌

4、使用令牌，使用`id_token`访问相关接口，`id_token`作为`Jnx-Access-Token`添加在header中 。

当令牌无效后，平台返回状态码为401，需要要重新请求令牌，在令牌过期之前可提前刷新令牌以延长令牌的有效期。

## 接口使用注意事项

1、设备上报相关数据时，都会携带公共参数，参见 【附件\>公共参数】

2、当业务接口返回结果为实体或传入参数为实体时以及设备上传的结构为实体时，可参见【附件\>具体表】理解每个字段的含义及枚举类型

3、如果无特殊说明，所有的重量单位都为公斤，所有的金额单位都为元,

金额为四舍五入，默认保留2位小数(0.105元 = 0.11元)，

重量为舍掉保留位之后的数字，默认保留2位小数 (0.105kg = 0.10kg)

4、设备可继承机构的配置，当需要统一配置参数时，将设备中的参数设置为继承机构，然后调用机构的相关接口进行设置。

5、设备、投口可独立配置，当需要独立配置参数时，参见独立设置相关接口进行设置。

6、不论是统一配置还是独立配置，都需要调用相应的创建配置接口生成新的配置。

7、当生成新的配置后，平台会自动处理下发工作，可通过设备日志进行查询是否下发成功。

# 配置第三方接收地址

当上述准备好之后，就可以配置第三方接收地址，可以机构中统一配置，需要在设备中置空以继承机构设置，或在设备中独立配置，

1、机构配置方法：机构为批量配置，依次点击【首页】\>【机构管理】\>【设备配置】打开机构的设备配置页面，填写【第三方服务器接口地址(HTTPS)】，先点击保存，再点击生成新的配置，

2、设备配置方法：设备为独立配置，依次点击【首页】\>【设备管理】\>查询出设备，点击查询出的【设备名称】打开设备详情页面，依次点击\>【配置编辑】\>【配置设置】打开设备设置页面，填写【第三方服务器接口地址(HTTPS)】 ，填写对接地址，先点击保存，再点击生成新的配置，

3、不论是从机构还是设备中生成配置都会下发到设备，生成后设备若在线，会立即下发到设备，若离线，待开机上线后会同步最新配置。

接口文档中的{{baseUrl}} 为第三方api地址，如果区分其它地址，默认都是在此地址后拼接业务地址进行请求，当指定了其它地址，优先使用其它的地址

    示例：
    post {{baseUrl}}/device_state

# 故障跟踪

1、设备的所有者、或设备管理员登录后可以点击头像打开配置界面，在本地调试设备。

2、设备本地的`/sdcard/jnx-log/`、`/sdcard/log/(部分设备存在)`目录存放了设备工作日志，可在平台下载日志进行查看分析,

如配置是否正确下发，命令是否发送到设备，重量上否已正常上传等。

当日志无法体现故障场景，无法协助排除故障时，请携带设备编号、具体的发生时间、操作流程、订单号等可以用来追踪的信息反馈给我们，以诊断故障原因，同时不要删除故障设备上的日志。

# 从v6升级到v7注意事项

1、从v6升级，在过度期间应使用不同的接口地址，避免返回数据结构不一至导致设备闪退，“启用v6模式上报”开关在7.59版本之后失效

2、接收设备数据时，原在url中携带的data已变更到body中，请更新接收姿势，

3、接收设备数据时，部份的参数字段名称发生了变更，需要重新构造接收实体，

4、与服务端对接的开门接口发生了变更，需要按新参数名称传参，并区分大小写，

5、与服务端对接的拍照接口，由设备控制参数`相机厂商或相机模型`决定使用`{baseUrl}}/capture_hk`还是`{{baseUrl}}/capture_ys`接收拍照参数，

6、原需要返回给设备的是json文本，现支持返回json对象。

7、原设备上报中包涵了实体对象，json对象，json文本，现统一上报为`json`对象。
