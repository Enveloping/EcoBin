# 清运数据上传

本接口适用于清运数据上传，收到数据后，必须按要求返回相应的信息， 为避免数据丢失，保持数据的完整性，设备会持续上报重复的一条记录，直到接收成功为止(返回对应的状态code=0)， 如果服务端不按要求返回，将会导致设备记录端积压 当发生断网或延时较高场景时，会启用离线分批次数据上传，为避免服务端压力过载，一次通常为1-2条，间隔1-10秒，直到上传完为止。

上传地址：

    POST {{baseUrl}}/clean_data  

上传的公共参数：见附件【公共参数】 上传的body参数：

| 序号 | 字段 | 说明 |
|:---|:---|:---|
| 1 | sn | 订单编号，同data中的sn |
| 2 | data | 清运数据，见附件【clean】表 |
| 3 | onlink | 1在线上传 2离线上传 |
| 4 | userId | 用户id |
| 5 | door | 当前投口的状态，不论离线上传还是在线上传，都取当前最新的值，见附件【door】表 |

    服务端必须返回code=0：

**重要：服务端必须返回body为json结构的文本，且code=0**

    当不确定是否返回正确时，可下载设备中的日志，查看日志中的返回结果是否被成功解析

后端返回：

``` json
{
  "code":0,
  "msg":"错误提示消息"
}
```

后端返回值说明：

| 序号 | 字段 | 说明 |
|:---|:---|:---|
| 1 | `code` | 0 成功 -1 失败 ,收到后就要返回0，否则设备会持续上报这条记录，会导致设备记录端积压 |
| 2 | `msg` | 错误提示消息 |
