# 相机拍照

拍照支持使用两个接口以区分不同的后端服务，在设备配置中修改`相机厂商`字段，设备根据设置的值决定请求哪个路径，

当内景有USB摄像机时，内部不会向后台拍照。

请求路径：

    POST  {{baseUrl}}/capture_hk   #  类型1 国标相机
    POST  {{baseUrl}}/capture_ys   #  类型2 莹石相机    

请求参数：

1、 header：

| 序号 | 字段           | 使用场景         |
|:-----|:---------------|:-----------------|
| 1    | `Content-Type` | application/json |

2、 params：

| 序号 | 字段     | 使用场景                |
|:-----|:---------|:------------------------|
| 1    | 公共参数 | 参见 【附件\>公共参数】 |

3、 body：

| 序号 | 字段 | 类型 | 必报 | 示例 | 说明 |
|:---|:---|:---|:---|:---|:---|
| 1 | sn | String | 是 | aa1ecdc26064b2807388256999960077 | 订单编码，订单唯一 32位的16进制字符串 v7.60+ ,原`orderSn`已弃用 |
| 2 | orgId | String | 是 | 1 | 平台中的机构id 同orgId 默认为1代表没有配置文件 |
| 3 | devSn | String | 是 | aa1ecdc26064b280 | 设备SN编码，设备唯一 |
| 4 | doorId | String | 是 | 1 | doorId |
| 5 | appid | String | 是 | wx9e638ba024ed7868 | 小程序appid 默认为小智环保，需要在机构配置中修改 |
| 6 | type1 | String | 是 | 1 | 1 投递 2清运 |
| 7 | type2 | String | 是 | 1 | 1 开门 2 关门 |
| 8 | type3 | String | 是 | 2 | 场景 1 内景，箱内照片 2 外景，箱外照片 |
| 8 | gbsn | String | 是 | 32131100001320007606 | 国标编码 32131100001320007606 |

{ "userId":1,"type2":2,"devSn":"14393d1087dd6b23","orgId":920,"type1":1, "sn":"14393d1087dd6b237655405453580017", "appid":"wx9e638ba024ed7868", "gbsn":"32131100001320007606", "type3":2,"doorId":1 }

后端返回：

``` json
{
  "code":0,
  "url":"",
  "photos":"",
  "msg":"错误提示消息"
}
```

后端返回值说明：

| 序号 | 字段     | 使用场景                                      |
|:-----|:---------|:----------------------------------------------|
| 1    | `code`   | 0 成功 -1 失败                                |
| 2    | `msg`    | 错误提示消息                                  |
| 3    | `url`    | 本次照片 用于设备端通过日志查阅错误           |
| 4    | `photos` | 订单关联的所有照片 用于设备端通过日志查阅错误 |

    如果使用自己的莹石相机：参考莹石的文档【设备抓拍图片】：
    https://open.ys7.com/help/687?h=api/lapp/device/capture
