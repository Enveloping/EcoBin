# 重量连续累加异常处理方法

# 故障原因

发生时间：2025-3-7 5时 至 2025-3-7 15时 之间

因后台未捕获正确的状态导致每上报一次重量累加一次

修复状态：已修复，清运后即可恢复正常

影响设备：未启用对接的AB设备，且在发生时间期间有离线记录上传的

# 处理方法：

方法一：清运后归零，如需要清运准确的，需要参照方法二操作

方法二：

1、升级到新版本

2、修改设备的理论重量、即时重量为正确的值

3、修改投口的理论重量、即时重量为正确的值

4、修改完成后保存

5、删除设备的db目录，让设备闪退后重新获取投口的重量。

6、验证是否正确
