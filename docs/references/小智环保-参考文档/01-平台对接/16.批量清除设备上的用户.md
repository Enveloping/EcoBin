# 批量清除设备上的登录用户

使用场景：后端禁用用户时，同时需要清除设备上的用户信息，避免离线登录。

清除后，用户再次登录会重新请求后端接口，需要确保后端不返回用户信息(即返回data=null)，或返回的用户信息中状态`status`字段为禁用

请求接口:

    POST https://device.zsgai.com/open_api/delete_for_dev_sn_list

支持版本：v7.66

开放时间：2025-2-25

请求参数：

1、 header：

| 序号 | 字段 | 使用场景 |
|:---|:---|:---|
| 1 | `Jnx-Access-Token` | 客户端请求时将`id_token`作为`Jnx-Access-Token`传递到服务端 |
| 2 | `Content-Type` | application/json |

2、 body：

| 序号 | 字段 | 格式  | 说明                                             |
|:-----|:-----|:------|:-------------------------------------------------|
| 1    | list | Array | 数组形式的设备编码列表 ，单次请求最大不超过200个 |

请求示例：

``` json
{
  "list": ["bb5b5729978bd1c3","bb5b5729978bd1c3"] 
}
```

平台返回：

| 序号 | 字段         | 格式   | 使用场景                    |
|:-----|:-------------|:-------|:----------------------------|
| 1    | `code`       | int    | 0 成功 -1 失败 401 令牌无效 |
| 2    | `msg`        | string | 错误提示消息                |
| 2    | `errorSum`   | int    | 错误数量                    |
| 2    | `successSum` | int    | 成功数量                    |
| 2    | `error`      | Array  | 错误详情                    |

返回示例：

``` json
{
  "code":0,
  "error":[],
  "successSum":1,
  "errorSum":0,
  "msg":"处理完成",
  "serverName":"device.zsgai.com",
  "request_user_id":171841,
  "request_refresh_code":null,
  "responseTime":43
}
```
