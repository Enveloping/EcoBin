# 登录设备、投递开门、清运开门

这个是多用途的接口，包括投递开门、清运开门、多投口时只登录不开门\
1、当设备有多个投口时，若不传投口号，默认为登录状态，否则应传入投口号，\
2、当设备只有一个投口时，登录后会默认打开第一个投口(1号投口)，\
3、投递开门应传入角色 role = 9 ，当有多个同时，需要用户选择或传入doorId ，\
4、清运开门应传入角色 role = 4 ，并在设备上输入垃圾袋编码，如果不在设备上输入应连同bagNo一同传入，\
5、如果传入的 loginName = root 或 userId = 1 即为超级管理员，登录后可控制或修改设备，\
6、当传入money、score时，会显示在界面上，位置是根据设备类型展示，部分设备不支持，\
7、设备的所有者、或设备管理员登录后可以点击头像打开配置界面，在本地调试设备。

接口地址：

    POST https://device.zsgai.com/open_api/delivery_open_door   

1、 header：

| 序号 | 字段 | 使用场景 |
|:---|:---|:---|
| 1 | `Jnx-Access-Token` | 客户端请求时将`id_token`作为`Jnx-Access-Token`传递到服务端 |
| 2 | `Content-Type` | application/json |

2、 body：

| 序号 | 字段 | 类型 | 长度 | 必填 | 说明 | 示例 |
|:---|:---|:---|:---|:---|:---|:---|
| 1 | devSn | String | 16 | 是 | 设备编码 | 115ae9792ef695e2 |
| 2 | doorId | Integer | 1 | 是 | 默认0 投口的编号 | 1 |
| 3 | userId | Long | 1-12 | 是 | 第三方平台中的用户Id | 1000000 |
| 3 | loginName | String | 1-32 | 是 | 第三方平台中的用户登录名称，唯一，不包涵特殊字符及汉字 同时用于比较当前用户与设备上登录的用户是否一至，占用投递 | root |
| 4 | status | Integer | 1 | 是 | 默认0 用户的状态 0正常 1停用 |  |
| 5 | role | Integer | 1 | 是 | 默认9 角色 9普通用户 5设备管理员 4清运人员 1超级管理员 |  |
| 6 | ext | String | 0-100 |  | 可选 扩展信息可携带不超过100个字符，记录上传时原样返回（存在订单的备注‘remarks’中） | {ex:1,bx:2},1111 |
| 7 | bagNo | String | 5 | 是 | 清运时需要有垃圾袋编码，固定5位纯数字 设备初始次使用的编码为 00001 | 00001 |

返回：

| 序号 | 字段   | 使用场景       |
|:-----|:-------|:---------------|
| 1    | `code` | 0 成功 -1 失败 |
| 2    | `msg`  | 错误提示消息   |

成功返回示例：

``` json
{
  "code": 0,
  "msg": "开门成功", 
  ... // 忽略
}
```

失败返回示例：

``` json
{
  "code": 401,  
  "msg": "令牌无效",  // 见具体的消息提示
  ... // 忽略
}
```

# 投递关门

POST <https://device.zsgai.com/open_api/delivery_close_door>

1、 header：

| 序号 | 字段 | 使用场景 |
|:---|:---|:---|
| 1 | `Jnx-Access-Token` | 客户端请求时将`id_token`作为`Jnx-Access-Token`传递到服务端 |
| 2 | `Content-Type` | application/json |

2、 body：

| 序号 | 字段 | 长度 | 必填 | 说明 | 示例 |
|:---|:---|:---|:---|:---|:---|
| 1 | devSn | 16 | 是 | 设备编码 | 115ae9792ef695e2 |
| 2 | doorId | 1 | 是 | 默认0 投口的编号 | 1 |
| 3 | userId |  | 是 | 用户的ID ，判断当前的操作是否同一人，避免恶意关门 | 1 |
| 4 | loginName |  | 是 | 用户的登录名称，判断当前的操作是否同一人，v7.74后，只验证`userId` | admin |

返回：

| 序号 | 字段   | 使用场景       |
|:-----|:-------|:---------------|
| 1    | `code` | 0 成功 -1 失败 |
| 2    | `msg`  | 错误提示消息   |

成功返回示例：

``` json
{
  "code": 0,
  "msg": "关门成功", 
  ... // 忽略
}
```

失败返回示例：

``` json
{
  "code": -1,  
  "msg": "设备不在线",  // 见具体的消息提示
  ... // 忽略
}
```
