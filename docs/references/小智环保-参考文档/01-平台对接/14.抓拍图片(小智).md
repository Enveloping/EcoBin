# 抓拍图片(小智国标平台) -测试中

前提条件：

1、设备控制-设备信息中，修改像机厂商为\[自主研发\]的小智国标平台，修改国标编码为相机中的国标编码(通常贴在机身外部)，修改后保存，并下发到设备

流程：

2、设备开门时向后端发起拍照请求，

3、后端收到请求后，调用此文档中的接口请求抓拍并存储照片。

此接口适用于接入到(sip.xzhb.com)小智国标平台的国标相机（自研），后端收到设备请求时，调用此接口进行相机抓拍，其他渠道的相机，请参照相关的相机厂商文档。

重要：若抓拍成功，平台只缓存5秒，后端自行下载存储

请求路径：

    POST https://sip.xzhb.com/open_api/capture    # 适用于接入到小智国标平台的国标相机 

请求参数：

1、 header：

| 序号 | 字段 | 使用场景 |
|:---|:---|:---|
| 1 | `Jnx-Access-Token` | 客户端请求时将`id_token`作为`Jnx-Access-Token`传递到服务端 |
| 2 | `Content-Type` | application/json |

2、 body：

| 序号 | 字段     | 类型   | 必报 | 示例                 | 说明            |
|:-----|:---------|:-------|:-----|:---------------------|:----------------|
| 1    | gbsn     | String | 是   | 32131100001320000001 | 国标编码        |
| 2    | deviceId | String | 是   | 32131100001320000001 | 国标编码 (临时) |

成功示例：

``` data

二进制照片数据，必须立即存储，平台5秒后删除，存储的扩展名为.jpg
```

失败示例：

``` json
{
  "code":-1,
  "devSn":"aa1ecdc26064b280",
  "doorId":"0",
  "type":"0",
  "sn":"aa1ecdc26064b2807388256999960077",
  "gbsn":"32131100001320000001",
  "msg":"抓拍失败"
}
```

后端返回值说明：

| 序号 | 字段   | 使用场景           |
|:-----|:-------|:-------------------|
| 1    | `code` | -1 失败            |
| 2    | `msg`  | 错误提示消息       |
| 3    | `sn`   | 订单编号，原样返回 |

    如果使用莹石相机：参考莹石官方的文档【设备抓拍图片】：
    https://open.ys7.com/help/687?h=api/lapp/device/capture
