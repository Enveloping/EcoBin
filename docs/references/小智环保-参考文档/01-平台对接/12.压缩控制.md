# 压缩机控制

# 接口说明

当配置有压缩机的设备，可以远程控制压缩机：开始压缩、停止压缩 由于压缩的状态与设备的各种状态处理比较复杂，需谨慎使用这个接口，不应开放给用户层，只建议在后台管理员操作。

接口地址：

    POST https://device.zsgai.com/open_api/compress_door

请求参数：

1、 header：

| 序号 | 字段 | 使用场景 |
|:---|:---|:---|
| 1 | `Jnx-Access-Token` | 客户端请求时将`id_token`作为`Jnx-Access-Token`传递到服务端 |
| 2 | `Content-Type` | application/json |

2、 body：

以下字段，用到哪个就携带哪个，为避免误操作不要全部携带，除非对设备的工作原理非常熟悉，当参数不生效或出现错误时，及时上传设备中的日志，看与设置的是否一至，检验是否成功下发。

| 字段 | 数据类型 | 非空 | 默认值 | 说明 |
|:---|:---|:---|:---|:---|
| devSn | Long | 必填 |  | 设备编号 |
| doorId | Long | 必填 |  | 投口编号 |
| cmd | int | 必填 |  | 0状态上报 1 开始压缩 2停止压缩 |
| all | int |  |  | 不要传值， 当有多个筒时，相当于选择所有投口，暂时不可用，后续会支持 |

平台返回：

``` json
{
  "code": 0 // 0成功 其它失败
  "msg": "发送成功",
  "...": "忽略"
}
```
