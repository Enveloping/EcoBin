# 获取令牌

为了更好理解工作流程，我们使用html+javascript写了一个demo【xzhb-v7-auth-demo.zip】演示如何获取令牌、调用投递开门接口以及刷新令牌，并已通过跨域测试\
可部署此demo查看工作原理，demo中敏感信息不应保存在前端，仅为演示目的。

此demo为静态文档，已部署在文档站点，点击右上角选择正式环境打开测试，调用逻辑代码可在开发者模式【F12】查看。

# 第一步 创建应用

登录平台，创建属于自己的应用，并获取appid和secret，应用创建之后，请联系客服开通应用的对接权限，每个应用都需要独立开通，

平台\>应用管理：<https://user.zsgai.com/jnx_app_client/list>

    示例：
    appid: 115ae9792ef695e2 
    secret: 8bc47ce763a86364ddb1477268eb9e21

| 序号 | 字段   | 长度 | 说明               | 示例                             |
|:-----|:-------|:-----|:-------------------|:---------------------------------|
| 1    | appid  | 16   | 应用ID `jnx_appid` | 115ae9792ef695e2                 |
| 2    | secret | 32   | 应用密钥           | 8bc47ce763a86364ddb1477268eb9e21 |

注意：变更`secret`后，原`jnx-access-token`或`id_token`会立即失效。

# 第二步 请求令牌（授权信息）

## 接口地址

请求方法：GET\
请求路径:

        GET https://user.zsgai.com/oauth2/client_authorize   

路径中参数: 拼接在路径中

| 序号 | 字段 | 长度 | 说明 | 示例 |
|:---|:---|:---|:---|:---|
| 1 | jnx-appid | 16 | 应用id `jnx_appid` | 115ae9792ef695e2 |
| 2 | signature | 32 | 签名信息 | 两次md5： String signature = Md5(Md5(appid + secret)); |

返回示例：

``` json
{
  "id_token":"*****", // 重要
  "refresh_code": "6381dedfa4ce09ede3a858de5c696cff", // 重要 
  "expires": 300000,  // 重要 令牌的有效期 秒 默认7天
  "iat": 1738917277,  // 创建时间 秒 时间辍/1000 
  "exp": 1738917307,  // 过期时间 秒 时间辍/1000 
  "code": 0, // 0 请求成功  -1 请求失败 401 鉴权失败     
  "msg": "获取成功", // 失败提示消息
  "responseTime":, 16 // 接口响应时间 毫秒
  "leeway": 60,  // 忽略 
  "userid": 84,  // 忽略 在第一次创建时可获取到,表示当前应用所属的用户id
  "username": "wx7337355831280002",
  ...  // 其它的信息忽略
}
```

返回值说明：

| 序号 | 字段 | 说明 |
|:---|:---|:---|
| 1 | `id_token` | 客户端请求时将`id_token`作为`jnx-access-token`传递到服务端，有效期 30天 |
| 2 | `refresh_code` | 当快过期时，用于刷新令牌 |
| 3 | `expires` | 过期时间（毫秒） 当前为7天 (60 *60* 24 *7* 1000) |
| 4 | `msg` | 提示消息 |
| 5 | `code` | 0 请求成功 -1 请求失败 401 鉴权失败 |
| 6 | `responseTime` | 接口响应时间 |

# 第三步 刷新令牌

当令牌快到期时，应及时刷新令牌，以延长令牌的有效期：

请求方法：POST\
请求路径:

    https://user.zsgai.com/oauth2/app_refresh_token   

header参数:

| 序号 | 字段 | 类型 | 示例 |
|:---|:---|:---|:---|
| 1 | `Jnx-Access-Token` | String | 值为第二步骤授权接口返回的`id_token` |
| 2 | `Content-Type` | application/json |  |

body参数:

| 序号 | 字段 | 类型   | 示例                                                  |
|:-----|:-----|:-------|:------------------------------------------------------|
| 1    | code | String | `refresh_code`（必须,第二步骤获取到的`refresh_code`） |

返回示例：同第二步

``` json
{
  "id_token":"*****", // 重要
  "refresh_code": "6381dedfa4ce09ede3a858de5c696cff", // 重要 
  "expires": 300000,  // 重要 令牌的有效期 秒 默认7天
  "iat": 1738917277,  // 创建时间 秒 时间辍/1000 
  "exp": 1738917307,  // 过期时间 秒 时间辍/1000 
  "code": 0, // 0 请求成功  -1 请求失败 401 鉴权失败     
  "msg": "获取成功", // 失败提示消息
  "responseTime":, 16 // 接口响应时间 毫秒
  "leeway": 60,  // 忽略 
  "userid": 84,  // 忽略 在第一次创建时可获取到,表示当前应用所属的用户id
  "username": "wx7337355831280002",
  ...  // 其它的信息忽略
}
```

返回值说明：同第二步

| 序号 | 字段 | 说明 |
|:---|:---|:---|
| 1 | `id_token` | 客户端请求时将`id_token`作为`jnx-access-token`传递到服务端，有效期 30天 |
| 2 | `refresh_code` | 当快过期时，用于刷新令牌 |
| 3 | `expires` | 过期时间（毫秒） |
| 4 | `msg` | 提示消息 |
| 5 | `code` | 0 请求成功 -1 请求失败 401 鉴权失败 |
| 6 | `responseTime` | 接口响应时间 |

# 第四步 请求相关业务接口

当获得id_token后就可以访问开放接口，POST请求时将参数添加到header中，根据不同的业务调用不同的URL，见具体的业务文档

请求方法：POST\
请求路径:根据不同的业务调用不同的URL，见具体的业务文档

header参数:

| 序号 | 字段 | 说明 |
|:---|:---|:---|
| 1 | `Jnx-Access-Token` | 客户端请求时将上步骤获取的`id_token`作为参数`Jnx-Access-Token`传递到服务端，有效期暂定15天 |
| 2 | `Content-Type` | application/json |

返回示例：根据不同的业务决定，参见具体文档

``` json
{
    "code":0  // 0 请求成功  -1 请求失败  401 鉴权失败
    "msg": "请求成功/失败提示消息" 
}
```
