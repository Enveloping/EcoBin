EcoBin 投递详细流程图 — 上传后建单 + 设备端继续投递

投递从「后端开门即建单」改为「设备上传称重后后端才建单」：用户在小程序「开启设备」只记录 该设备当前活跃用户会话（不建单）；设备投放称重后上报，后端此刻才建单，归属取活跃会话用户。 用户可在设备屏「继续投递」本地再开门投下一袋，仍归同一活跃用户、每袋独立成单。照片由设备 直传腾讯云 COS（位置由设备自定），并把 4 个 URL 随 deliveryComplete 回传，后端原样存。全程免公网。 开启设备·建会话 上传后建单 设备↔COS 直连

关键规则速查

点说明
身份关联 biz_device_session 按 device_id 唯一，一台设备一行「当前活跃用户」。开启设备 upsert；上传时 findActive 取归属。 后来用户开启会覆盖上一行（接受「最近用户」语义，交叠极罕见不额外处理）。TTL 默认 15 分钟，投递成功续期。
建单时机 设备上传称重之后才建单（不预建）。首投 / 续投对称，每袋独立成单（deliveryStatus=1 上传即完成）。
幂等键 OneNet 消息 id 分发器取报文消息 id（缺失回退 MQ messageId），按 device + 消息 id 幂等去重，落 biz_delivery_order.delivery_token 列（唯一索引兜底，适配 MQ at-least-once）。直连未带消息 id 则不去重。
照片 key 由设备自定位置（建议 {sn}/delivery/{唯一串}/<slot>.jpg），slot ∈ open_outside/open_inside/close_outside/close_inside。 开门只下发凭证（设备整会话缓存复用）；设备直传后把 4 个 URL 随 deliveryComplete 回传，后端原样写订单 photo_*（不复原）。
返现入账 命中活跃用户时 amount = door.price × weight（HALF_UP 2 位）入用户余额，与建单同事务；无主单不返现。
免公网 出站 开门下发 = 后端→OneNet 出站；事件上行 = 后端作 MQ 消费者拨出去；照片 = 设备↔COS 直连。无任何「设备/平台 → 后端」入站端点。

对应代码：开启设备 AppDeliveryController.openDoor → DeliveryOrderServiceImpl.openDoor（建会话 + 下发）； 下发 DeviceCommandService.sendOpenDoor(sn,doorIndex) → OneNetClient.openDeliveryDoor（cosToken 仅凭证）； 上传后建单 OneNetEventDispatcher.handleDeliveryComplete（或直连 IotDeliveryController.complete）→ DeliveryOrderServiceImpl.completeDelivery； 会话 DeviceSessionService（迁移 V13__add_device_session.sql）。字段口径见 onenet-thing-model.md §3.1 / §4.1 / §8。
